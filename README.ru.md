# Job Finder Kit — поиск работы с помощью ИИ

Представьте **тетрадь на компьютере**: в одной таблице — вакансии, в папках — черновики писем и резюме. Рядом сидит **помощник в программе Cursor** (умный чат): он ищет вакансии, пишет черновики, вы всё проверяете и правите. **Врать или придумывать опыт нельзя** — только ваши реальные факты.

Программистом быть не нужно. Нужны: Cursor, интернет и готовность один раз заполнить свои данные.

---

## Как скачать

**С GitHub:**

```bash
git clone https://github.com/nikita-gorban/find-me-a-job.git
cd find-me-a-job
```

**Или** на GitHub: **Code → Download ZIP** → распаковать → открыть папку в Cursor.

Папки с вашими откликами в `applications/` **не попадают в git** — черновики и PDF остаются только у вас на компьютере.

## Как это работает (очень просто)

1. **Помощник** находит вакансии и записывает их в файл [tracker.md](tracker.md) (таблица с номерами).
2. **Вы** говорите: «нравится №3 и №7».
3. **Помощник** создаёт в папке `applications/` черновики: описание вакансии, сопроводительное письмо, иногда отдельное резюме.
4. **Вы** читаете, исправляете слова, говорите «ок» или «перепиши короче».
5. Если нужен **PDF-файл резюме** (не на hh.ru) — одна команда в терминале (в ONBOARDING объяснено).
6. **Вы** сами нажимаете «откликнуться» на сайте работодателя.

Помощник **не** отправляет отклики за вас и **не** гарантирует оффер. Он экономит время на рутине.

## Два типа сайтов

| Где ищете | Что готовим в папке | Где резюме при отклике |
|-----------|---------------------|-------------------------|
| **hh.ru** | Письмо + заметки | Резюме уже на hh (вы копируете из [cv-base-ru.md](cv-base-ru.md)) |
| **Другие** (письмо на email, Lever, Greenhouse…) | Письмо + файл резюме + при необходимости PDF | Прикрепляете PDF или текст сами |

## Что лежит в папке (простыми словами)

| Файл | Зачем |
|------|--------|
| **[ONBOARDING.md](ONBOARDING.md)** | Первый запуск по шагам — **начните здесь** |
| **[AGENTS.md](AGENTS.md)** | Правила для помощника: какие вакансии искать, как писать письма. **Заполнить под себя** |
| **[CURSOR.md](CURSOR.md)** | Как открыть чат, что нажать |
| **cv-base-en.md** | Ваше главное резюме (англ.) — отсюда всё копируется |
| **cv-base-ru.md** | То же по-русски, в т.ч. для hh |
| **tracker.md** | Список вакансий |
| **generate_cv_unified.py** | Маленькая программа: из текста резюме делает один PDF |
| **папка applications/** | Отдельная папка на каждую вакансию |
| **applications/_example-do-not-track/** | Пример «как должно выглядеть» — в список вакансий не пишем |

Файлы с окончанием `.md` — обычный текст, открывается в Cursor как в блокноте.

## Если пришёл zip-архив

1. Распакуйте (двойной клик).

3. Выберите папку, **внутри которой сразу видны** `README.ru.md` и `AGENTS.md`.  
   Если видите только ещё одну папку внутри — зайдите в неё и откройте уже её.

## Статусы в таблице (что значат слова в tracker)

| Слово | По-человечески |
|--------|----------------|
| `new` | Нашли, ещё не готовили отклик |
| `cv_draft` | Черновик резюме под вакансию — надо проверить |
| `cv_approved` | Резюме ок |
| `cover_done` | Сопроводительное письмо ок |
| `pdf_ready` | PDF собран |
| `applied` | Вы уже откликнулись |
| `skip` | Решили не откликаться — потом строку можно убрать |

## Важно помнить

- Помощник иногда ошибается — **всегда читайте** перед отправкой.
- Первое сообщение в чате лучше начать с **`@AGENTS.md`** — так он точно прочитает ваши правила.
- Сопроводительное — **короткое**, 3–5 предложений, на **языке текста вакансии**.
- Python и PDF нужны **не всем** — только если откликаетесь не через hh и нужен файл PDF. Подробности в ONBOARDING.

Удачи. Если застряли на шаге — спросите помощника в чате: «Я на шаге X из ONBOARDING, не получается …».
